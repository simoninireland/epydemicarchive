Welcome

This is the epydemic archive, a repository of random (and other) networks for doing network science.

Why does the archive exist?

Network science uses random networks to test how processes behave under different conditions, without constraining experiments to a single network that might have specific features that affect the process. This means that a typical experiment makes use of a lot of networks: we might perform an experiment at 100 points in a parameter space and conduct 100 repetitions at each point to get decent averaging. That would require 10,000 random networks if we conducted each repetition on its own network; even if we're more parsimonious we might still need several hundred networks to ensure fairness.

This isn't a problem for small networks and for some topologies: Erdős-Rényi networks are cheap to construct, even for large orders. But more complex topologies can become expensive to generate, and this slows down experiments.

Wouldn't it be useful to be able to pre-create a library of networks with different sizes and topologies, and then re-use them from simulations? We could then separate the time spent generating networks from the time spent using them for experiments. Of course we'd have to be careful that we had enough networks to choose from, and that we didn't spoil out statistics by accidentally using too few different networks.

The epydemic archive performs this task.

How do I use the archive?

To download networks from the archive, you can either browse the entire collection or search the archive against tags or criteria such as size, degree distribution, and so on. You can also submit networks in a range of common file formats. Accessing the archive is unrestricted; to upload networks you need to register and then sign in.

However, this manual use of the archive is inconvenient for large-scale use. An easier way is to access the archive programmatically using its API. If your work makes use of the epydemic simulation framework then the archive can be used transparently as a source of networks for simulations, and you can also submit networks directly from programs.
